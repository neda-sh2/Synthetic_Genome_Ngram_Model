# Generating Synthetic Genomes with N-gram (Markov Chain) Models

## Overview
This project applies N-gram modeling — a technique from natural language processing — to genomic sequence data. By treating k-mers (short DNA subsequences) as the genomic equivalent of "words," a Markov chain model is trained on a real bacterial genome to learn which nucleotide tends to follow each k-mer context. The trained model is then used to generate entirely synthetic DNA sequences, which are statistically validated against the real genome.

## Biological Question
Can a simple, local statistical model (no knowledge of genes, codons, or regulatory elements — just "what nucleotide tends to follow this short sequence?") generate synthetic DNA that mirrors the sequence composition of a real genome?

## Data
- **Source genome:** bacterial genome assembly, NCBI accession `NZ_ACYS00000000`, provided as a `.tar` archive containing a FASTA scaffold file

**Note:** the genome file is not included in this repository and is not currently hosted for automatic download. To run this notebook, you'll need to either:
1. Download the assembly directly from NCBI using the accession number above, or
2. Upload your own bacterial genome FASTA file and adjust the filename in the notebook accordingly

## Methods & Tools

**Language:** Python 3.10+ (standard library only — no external ML/bioinformatics packages required for the core model)

| Library | Purpose |
|---|---|
| `tarfile` / `os` | Genome archive extraction and file handling |
| `collections` (`defaultdict`, `Counter`) | Building transition count tables and k-mer frequency distributions |
| `random` | Sampling from learned probability distributions |
| `math` | Jensen-Shannon divergence calculation |
| `matplotlib` | K-mer frequency comparison visualization |

**Pipeline:**
1. Parse and clean a real genome assembly (FASTA format)
2. Build a k-mer → next-nucleotide transition count model (k=3 by default)
3. Normalize counts into transition probabilities
4. Generate multiple synthetic genome sequences by sampling from the trained model
5. Export synthetic sequences in FASTA format
6. Validate synthetic sequences against the real genome using:
   - GC content comparison
   - K-mer frequency distribution comparison
   - Jensen-Shannon divergence (a formal, bounded similarity metric)

## Key Results
- Synthetic genomes generated by the model closely match the real genome's GC content
- K-mer frequency distributions between real and synthetic genomes show strong agreement, particularly for the most common k-mers
- Jensen-Shannon divergence between real and synthetic k-mer distributions is low, confirming the model successfully captures local sequence composition statistics

**Important caveat:** low divergence in k-mer frequency does not imply the synthetic genome is biologically functional. This model captures only local statistical structure — it has no representation of genes, open reading frames, or regulatory elements.
